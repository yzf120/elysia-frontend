[//]: # (## 🔍 问题原因)

[//]: # ()
[//]: # (错误信息：`index.js:1 Failed to load resource: the server responded with a status of 404 &#40;Not Found&#41;`)

[//]: # ()
[//]: # (**根本原因**：)

[//]: # (- Vite 配置中有一个 `/api` 代理规则，将所有 `/api/*` 请求代理到后端服务器 `http://localhost:8001`)

[//]: # (- 前端代码中有 `/api/index.js` 文件（前端 API 服务层）)

[//]: # (- 当浏览器尝试加载 `/api/index.js` 时，Vite 将请求代理到后端，但后端没有这个文件，返回 404)

[//]: # ()
[//]: # (**冲突示意**：)

[//]: # (```)

[//]: # (浏览器请求: /api/index.js)

[//]: # (    ↓)

[//]: # (Vite 代理规则: /api/* → http://localhost:8001/api/*)

[//]: # (    ↓)

[//]: # (后端服务器: http://localhost:8001/api/index.js &#40;不存在&#41; → 404)

[//]: # (```)

[//]: # ()
[//]: # (---)

[//]: # (**工作原理**：)

[//]: # (1. 浏览器发送请求到：`http://localhost:3000/api/xxx`)

[//]: # (2. Vite 检测到 `/api` 前缀)

[//]: # (3. Vite 将请求转发到：`http://localhost:8001/api/xxx`)

[//]: # (4. 后端处理请求并返回响应)

[//]: # (5. Vite 将响应返回给浏览器)

[//]: # ()
[//]: # (**注意事项**：)

[//]: # (- ⚠️ 前端代码文件不能放在 `/api/` 目录下)

[//]: # (- ⚠️ 只有运行时的 HTTP 请求会被代理，静态文件加载不会被代理)

[//]: # (- ✅ 使用 `/services/` 或其他名称存放前端 API 服务层代码)
