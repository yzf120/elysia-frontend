# 🚀 快速开始指南

## 📋 前置条件

1. **后端服务已启动**
   ```bash
   cd /Users/sylvainyang/project/elysia/elysia-backend
   go run main.go
   ```
   后端服务应运行在：`http://localhost:8001`

2. **Node.js 环境**
   - Node.js >= 16.0.0
   - npm >= 8.0.0

## 🎯 启动步骤

### 方式一：使用启动脚本（推荐）

```bash
cd /Users/sylvainyang/project/elysia/elysia-frontend/app-frontend
./start.sh
```

### 方式二：手动启动

```bash
cd /Users/sylvainyang/project/elysia/elysia-frontend/app-frontend

# 首次运行需要安装依赖
npm install

# 启动开发服务器
npm run dev
```

## 🌐 访问应用

启动成功后，在浏览器中访问：

**http://localhost:3001**

> 💡 提示：建议使用 Chrome 浏览器的移动设备模拟模式（F12 → 切换设备工具栏）

## 🔍 功能验证

### 登录成功后

1. **查看用户信息**
   - 打开浏览器开发者工具（F12）
   - 切换到 Application/存储 标签
   - 查看 Local Storage
   - 应该能看到：
     - `token`: JWT 令牌
     - `userInfo`: 用户信息（JSON）
     - `userType`: 用户类型（admin/teacher/student）

2. **测试主页功能**
   - 查看用户头像和信息

   - 应该自动跳转到首页

## ⚠️ 注意事项

### 1. 验证码功能
- 需要后端配置短信服务
- 如果未配置，只能使用密码登录
- 开发环境可以查看后端日志获取验证码

### 2. 教师注册
- 需要管理员审核
- 注册后无法立即登录
- 需要管理员在后台审核通过

### 3. 端口冲突
- 默认端口：3001
- 如果被占用，修改 `vite.config.js` 中的端口

### 4. CORS 问题
- 后端已配置 CORS 中间件
- 如果仍有问题，检查后端服务是否正常运行

## 🐛 问题排查

### 页面空白
1. 检查浏览器控制台是否有错误
2. 确认依赖已正确安装：`npm install`
3. 清除浏览器缓存并刷新

### API 请求失败
1. 确认后端服务运行在 `http://localhost:8001`
2. 检查网络请求的状态码和响应
3. 查看后端日志

### 登录失败
1. 确认账号密码正确
2. 检查网络请求是否成功
3. 查看错误提示信息

### 验证码收不到
1. 检查后端是否配置短信服务
2. 查看后端日志中的验证码
3. 确认手机号格式正确
